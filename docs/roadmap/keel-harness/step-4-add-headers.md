# Step 4: Add MPL-2.0 headers to harness source

Prepend the exact MPL-2.0 header to every `.ts`, `.tsx`, `.js` file in `packages/ai` and `packages/agent` that lacks it. Modify `generate-models.ts` to emit the header in generated output.

---

## Context

MPL-2.0 adoption (Decision 3) requires every source file in the harness to carry the license header. Current ai and agent source files have NO MPL headers. `packages/ai/src/models.generated.ts` is generated by `packages/ai/scripts/generate-models.ts` at build time; adding a header to the generated file in this step would be wiped by `npm run build` in Step 6. The generator template itself must be modified.

## Decision

1. Find every `.ts`, `.tsx`, `.js` file under `packages/ai` and `packages/agent` using `grep -rL "Mozilla Public License"` to exclude files that already have it.
2. Prepend the exact header text (see below) to each file via a bash loop with `sed -i`.
3. Modify `packages/ai/scripts/generate-models.ts` line ~1606 to include the MPL-2.0 header in the generated `output` string, replacing the current `// This file is auto-generated...` comment.

Exact header text:
```
/* This Source Code Form is subject to the terms of the Mozilla Public
 * License, v. 2.0. If a copy of the MPL was not distributed with this
 * file, You can obtain one at https://mozilla.org/MPL/2.0/. */
```

## Constraints

- Must not double-header files that already have it.
- Must not corrupt binary files (filtered to text extensions only: `.ts`, `.tsx`, `.js`).
- Must not break file encoding (sed handles UTF-8/LF correctly; `.gitattributes` enforces LF).

## Second-order effects

- ~118 files will be modified (verified via `find` count). This is a bulk operation.
- The commit will be large but purely additive — no logic changes.
- After `npm run build` in Step 6, `models.generated.ts` will be regenerated WITH the MPL-2.0 header, satisfying FL-4.
- Future hull builds must preserve these headers when repackaging.

## Failure modes

- *Deceptively obvious wrong answer*: "Use a single sed command on all files." Wrong: files with different line endings could be corrupted. The bash loop with `grep -rL` exclusion is safer.
- Double-headering files that already have it. Fix: `grep -rL "Mozilla Public License"` excludes files with existing headers.
- `npm run build` regenerates `models.generated.ts` without a header, wiping any manually-added header. Fix: Modify `generate-models.ts` template to emit the header in generated output.

## Alternatives considered

- Add header only to entry-point files. **Rejected**: MPL-2.0 best practice is every file. GitHub's license detection and legal defensibility require it.
- Use an npm package like `license-header`. **Rejected**: Adds a dependency for a one-time operation. Bash loop with sed is simpler.

## Self-consistency check

Consistent with Decision 3 (MPL-2.0 LICENSE file). The header references the same license. Both decisions serve the same goal of legally protecting the harness.

## Files touched

- All `.ts`/`.tsx`/`.js` files under `packages/ai/src/`, `packages/ai/test/`, `packages/agent/src/`, `packages/agent/test/`
- `packages/ai/scripts/generate-models.ts` — modify `output` template string to emit MPL-2.0 header

## Acceptance criteria

- AC-4.1: Zero `.ts`/`.tsx`/`.js` files in `packages/ai` or `packages/agent` lack the MPL-2.0 header (verified by `grep -rL "Mozilla Public License" packages/ai packages/agent --include="*.ts" --include="*.tsx" --include="*.js"` returns empty)
- AC-4.2: No file has a double header (verified by checking no file contains the header text twice)
- AC-4.3: `npm run build` still passes after header insertion (headers are valid JS/TS comments)
- AC-4.4: `packages/ai/scripts/generate-models.ts` output template contains the MPL-2.0 header before `export const MODELS`

## Verification commands

```bash
# AC-4.1
grep -rL "Mozilla Public License" packages/ai packages/agent --include="*.ts" --include="*.tsx" --include="*.js" | wc -l | grep -q '^0$'

find packages/ai packages/agent -name "*.ts" | while read f; do

done
# AC-4.3
npm run build
# AC-4.4
grep -q "This Source Code Form is subject to the terms of the Mozilla Public" packages/ai/scripts/generate-models.ts
```
